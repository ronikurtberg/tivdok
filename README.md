## מה זה תבדוק ולמה זה שונה מכל דבר אחר

### הבעיה האמיתית

כשאתם מוכרים רכב בישראל, אתם מסתכלים ביד2 ומנסים להבין מחיר בעצמכם.
אבל יד2 היא רק **פלטפורמה אחת** — ורובה לא מסוננת:

- מודעות ללא מחיר (מחכות שתתקשרו)
- מודעות ללא קילומטראז' (מה יש להסתיר?)
- מחירים גבוהים בצורה קיצונית (עיוות הממוצע כלפי מעלה)
- רכבי סוחרים שמעלים מחירים מלאכותיים
- אין נתוני Facebook Marketplace, AutoTrader, יד שנייה, ולוחות נוספים

**אתם מקבלים תמונה חלקית ורועשת, ומנסים לתמחר ממנה.**

### מה תבדוק עושה אחרת

תבדוק מבצע **ניתוח שוק עם מסנן חכם**:

1. **סורק מאות מודעות** ביד2 בזמן אמת (לא נתונים מיושנים)
2. **מסנן אוטומטית** מודעות ללא מחיר, ללא ק"מ, וחריגות קיצוניות
3. **מציג לכם את כל המתחרים האמיתיים** — כל רכב, כל מחיר, ממי
4. **משווה לכם לעומת המחיר הרשמי של משרד התחבורה** (מחיר קטלוג)
5. **מסביר כל החלטה** — המחיר המוצע מנומק, לא מופיע סתם
6. **יועץ AI שמכיר את הרכב שלכם** — לא ChatGPT גנרי, אלא מערכת שטוענת את כל הנתונים שלכם ומוכנה לענות על כל שאלה ספציפית

## מודל עסקי — ₪50 לדוח שנשאר איתכם לנצח

### הצעת הערך

> **₪50 — דוח שוק מלא לרכב שלכם, נשמר לכם לחיים.**

- הדוח קשור למספר הרכב — לא לחשבון אישי
- בעלים הבא של הרכב יכול לגשת לדוח (מעדכן עם נתוניו)
- כל שינוי בפרטים, מחיר, תמונות — מתעדכן בפרופיל
- לאורך זמן: "כמה שווה הרכב שלי *היום*?" — שאלה שאפשר לבדוק תמיד

### פרופיל רכב לכל רכב בישראל

**החזון:** כל רכב בישראל מקבל דף פרופיל בתבדוק.
- מספר רכב = כתובת קבועה (כמו domain)
- מכיל: נתוני רישוי רשמיים, היסטוריית טסטים, ניתוח שוק, תמונות, תיאור
- עובר בין בעלים — הבעלים הבא יכול לעדכן ולהמשיך
- יוצר **מאגר שקיפות לשוק הרכב הישראלי** — המקום שבו מחירים מוסברים, לא נובעים מהשערה

### מסלולים עתידיים

| מסלול | מה כולל | מחיר |
|-------|---------|------|
| **דוח בסיסי** | ניתוח שוק + מחיר מוצע + פרופיל רכב | ₪50 |
| **דוח פרמיום** | + ניקוד תמונות AI + תוכנית מכירה מפורטת + יועץ AI ללא הגבלה | ₪120 |
| **מנוי מוכר** | דוחות ללא הגבלה + עדכוני שוק חודשיים | ₪29/חודש |

## שאלה טכנית: למה צריך PDF?

### מה מחזיר ה-API הממשלתי

data.gov.il מחזיר עבור כל מספר רכב (resource `053cea08`):

| שדה | מה זה |
|-----|-------|
| ✅ שנה, יצרן, דגם מסחרי | תמיד זמין |
| ✅ צבע, סוג דלק | תמיד זמין |
| ✅ גרסה (trim), VIN | תמיד זמין |
| ✅ תאריך טסט אחרון, תוקף רישיון | תמיד זמין |
| ✅ סוג בעלות (פרטי/מסחרי) | תמיד זמין |
| ❌ נפח מנוע (cc) | **לא קיים בריסורס הזה** |
| ❌ מספר דלתות | **לא קיים** |
| ❌ סוג מרכב (סדאן/האצ'בק) | **לא קיים** |
| ❌ יד (מספר בעלים) | **לא קיים — רק "פרטי/מסחרי"** |
| ❌ עיר | **לא קיים** |

### איך יד2 ואחרים מקבלים נפח מנוע ודלתות?

יד2 ושאר האתרים ככל הנראה משלבים שני מקורות:
1. **ריסורס הרישום** (053cea08) — לנתוני הרכב הספציפי
2. **קטלוג הדגמים** (degem-rechev-wltp ו-resources נוספים) — לנתוני מפרט לפי קוד דגם (`degem_cd`)

חיבור בין `degem_cd` מהרישום לטבלת המפרטים מאפשר לקבל cc, דלתות, סוג מרכב.
**זה TODO בתבדוק — implement second API lookup via degem_cd.**

### כרגע
- מספר רכב בלבד → נתונים בסיסיים (שנה, יצרן, דגם, גרסה, VIN, טסט)
- PDF של רישיון רכב → **כל הנתונים** (cc, דלתות, ק"מ, יד, עיר, סוג מרכב)
- הפתרון לטווח קצר: לבקש מהמשתמש להזין cc/דלתות ידנית (כבר קיים)
- הפתרון לטווח בינוני: lookup שני דרך degem_cd

## הרצת הפרויקט

### 1. התקנה (פעם ראשונה)

```bash
cd ~/work/car-seller-assistant
uv pip install -e .

cd frontend
npm install
```

### 2. Backend (FastAPI — port 8000)

```bash
cd ~/work/car-seller-assistant
.venv/bin/python -m uvicorn car_seller.api:app --host 0.0.0.0 --port 8000 --reload
```

### 3. Frontend (React + Vite — port 5173)

```bash
cd ~/work/car-seller-assistant/frontend
npm run dev
```

פתחו **http://localhost:5173**

### 4. מפתחות AI

```bash
cp .env.example .env
# הוסיפו:

# ANTHROPIC_API_KEY=sk-ant-... (Claude Haiku — אופציונלי, משתמש בוחר ב-UI)
```

## נקודות API

| Method | Path | תיאור |
|--------|------|-------|
| `GET` | `/api/plate/{plate}` | נתוני רכב ממשרד התחבורה |
| `GET` | `/api/history/{plate}` | היסטוריית טסטים שנתיים |
| `POST` | `/api/analyze` | סריקת יד2 + מחיר קטלוג + תוכנית מכירה |
| `POST` | `/api/chat` | יועץ AI (GPT או Claude, עם כל הקשר הרכב) |
| `POST` | `/api/score-photos` | ניקוד תמונות רכב עם מודל Vision |
| `POST` | `/api/parse-license` | פרסור PDF של רישיון רכב |
| `GET/POST/DELETE` | `/api/cars` | שמירת/עריכת/מחיקת רכבים |
